Five open decisions — recommended choices

Each card lists the options from the brief; the highlighted one is the default we've drawn through the rest of the document. Don, swap the highlight to redirect the manifest.

Decision 1 · Tablet landscape layout

Recommended
right-side-drawer

Visualizer 60% / panel 40%, panel on the right edge.

  • Visualizer stays large & intact for the tech
  • Panel has room for full turn rows & multi-line input
  • Reads as a chat — familiar, no learning
Alt
bottom-strip

Visualizer top 60% / panel bottom 40%.

  • Better for portrait, awkward in landscape
  • Crops nail visualizer vertically
Alt
split-30pct

Visualizer 70% / panel 30%.

  • Tightest text column — wraps at ~28ch
  • Input feels cramped at 500-char cap

Decision 2 · IterationTimeline separation

Recommended for slice 7
collapse-into-panel

Panel scroll IS the history. No separate timeline component.

  • Cheaper — drops conditional manifest item #8
  • Single source of "where am I in the session"
  • Defer separate component until user testing demands it
Alt — see §06
separate-component

Compact 60×60 thumbnail strip in visualizer area.

  • Best when paired with decision 4 = user-selectable
  • Adds duplication of state across two surfaces

Decision 3 · Orphaned-turn failure UX

Recommended
inline-error-badge-retry

Failed turn keeps its row, badge swaps to red "Failed", inline Retry button.

  • Message visibly persisted — no orphan illusion
  • Retry sits where the user is already looking
  • Multiple failures don't fight for one banner slot
Alt
system-message-retry

Neutral row + inserted system message below.

  • Adds a message type the rest of the app doesn't have
Alt
dismissible-banner

Full-width banner above panel.

  • Detaches the error from the failing turn
  • One banner can't represent two failures

Decision 4 · Current-image indicator

Recommended for slice 7
auto-latest-badge

Latest successful turn auto-shows a "Current" badge + accented border.

  • Zero new affordances — fits the slice budget
  • Matches mental model "newest = on screen"
  • Leaves room for user-selectable in slice 8+
Alt — future slice
user-selectable

Tap any successful turn to load its image.

  • Enables A/B comparison — strong UX value
  • Pairs naturally with separate IterationTimeline
  • Adds state mgmt: "viewing turn N" ≠ "latest"

Decision 5 · Phone scope

Recommended
tablet-only

Below ~640px viewport, show a "best on tablet" notice w/ link back.

  • Project profile says tablet-first — phone is "if cheap"
  • Refinement chat + visualizer + 500-char input is too much for 390px
  • Saves slice 7 budget; revisit if usage data shows phone demand
Alt — costlier
first-class-phone

Single-column collapse: visualizer fixed top, panel scrolls below.

  • Adds another responsive break to test
  • Sketched in §08 as a fallback — not the default

Open notes for Don

  • Insertion point: All wireframes mount the panel inside the existing Confirm.tsx success branch — directly after VisualizerFrame, directly before the "‹ Back to adjust" button. The "Back to adjust" CTA stays visible in every state (footer row).
  • Where motion will help (deferred): turn-row state transitions (pending → success | failure), visualizer image swap on new success, "Current" badge moving to the latest successful turn, and the input-area transition when entering the session-full terminal. Choreography to be specified in slice 7.5 / animation pass.
  • Empty whitespace input: submit is disabled in every active variant; visible in the input footer ("Enter to send · Shift+Enter for newline"). Char counter exists in all variants; warn styling kicks in at 450/500 (sketched as the accent color).
  • Brand tokens used: --nta-primary (current/submit), --nta-success (success badge), --nta-destructive (failure badge + retry), --nta-accent (session-full notice + warn counter), neutral surfaces and borders for the rest. No new colors.
  • Out of scope (per brief): animation timing, final type / color tweaks, implementation code, backend.
