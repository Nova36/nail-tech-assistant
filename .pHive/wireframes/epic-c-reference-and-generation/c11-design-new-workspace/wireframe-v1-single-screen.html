Studio New Design workspace.

Five state frames stacked top → bottom. Each uses the authenticated shell (sidebar + main) and the same component primitives: Browse Pinterest, Reference panel, Upload tile, Prompt, Generate bar.

Tokens
bg-background, bg-card, text-foreground, text-muted-foreground, bg-primary, bg-secondary, bg-accent, destructive. Heading: Fraunces. Body: Inter.
Frame conventions
Frames are 1280×variable. Window-bar shows route + frame label. Dashed pin-labels mark named regions / components. Callouts are interaction / a11y / motion notes.
Reference rules
1 primary required to enable Generate. 0–N secondaries optional (display order = generation weight). Exactly one primary at any time. Empty primary + ≥1 secondary = invalid (frame 4).

Keyboard reorder.

Detail · ReferenceSlot · grip:focus-visible

When a reorder grip receives keyboard focus (e.g. via Tab), an inline keyboard-hint chip surfaces immediately to the right. Pressing ↑/↓ moves the secondary up/down within the secondaries list (primary cannot be reordered). Each move fires an aria-live announcement.

Interactions

  • Pin click → adds as secondary · 200ms slide-in · toast 3s · aria-live polite.
  • Hover overlay → "Set as primary" affordance · 160ms fade.
  • Upload → drag-drop OR keyboard-activate the tile.
  • Reorder → drag (mouse) OR ↑/↓ (keyboard) · live during drag · 150ms ease drop.
  • Remove → 150ms fade-out · neighbors reflow 200ms · focus moves to next slot.
  • Generate disabled → reasons: "Select at least one primary reference." · "Pick exactly one primary."
  • Generate pressed → saving state · redirects to /design/[id] (out of scope here · see c17).
