Studio New Design — wizard refactor.

Same route, same brand tokens, same a11y/motion conventions — the workspace now flows in three steps. Step 1 — Inspiration · Step 2 — Direction · Step 3 — Confirm (separate wireframe). Wizard state is internal; the URL never changes. ← Back preserves all selections.

Tokens (unchanged)
bg-background, bg-card, text-foreground, text-muted-foreground, bg-primary, bg-secondary, bg-accent, destructive. Heading: Fraunces · Body: Inter.
Wizard chrome
Top progress strip on every frame: 1 Inspiration → 2 Direction → 3 Confirm. ✓ on completed steps (clickable to revisit, preserves state). Footer row: ← Back (hidden on Step 1) · Continue → (Step 1) / Generate → (Step 2).
Frame conventions
Frames 1280×variable. Six total: 1.1, 1.2 · 2.1, 2.2, 2.3, 2.4. Dashed pin-labels mark named regions. Callouts are interaction / a11y / motion notes.
Step boundary semantics
Step 1 = "this is interesting" — no primary marking, no order, soft cap of ~8 (warning, not block). Step 2 enters with all Step 1 picks locked as the working set; Pinterest browse + upload tile are hidden. Removing in Step 2 returns the item to "available" (re-includable via ← Back) — never deletes from app state.
Motion (additions)
Inter-step transition: 250ms slide-and-fade — left for next, right for back. prefers-reduced-motion: instant, no slide. All previously-noted 200ms slide+fade adds, 150ms fade-outs, 200ms reflow, and 150ms reorder still apply.
