Wireframe · c17 · /design/new · Step 3

Step 3 · Confirm.

Seven frames: pending → success, plus five distinct error states (refusal · rate-limit · network · storage-fail · low-quality). Same wizard chrome as c11 (sidebar + progress strip). Calm, recoverable language. No alarming red, no shake.

Tokens (locked)
Fraunces heading · Inter body · semantic Tailwind tokens. Error glyphs use primary-tint + accent + muted strokes — no flat red, no exclamation triangles.
Card geometry
All errors share the same 600px card shape: glyph (132×132) → headline → body → actions. Pending uses a 540px stage. Success breaks the card to give the image a 720px hero frame.
Progress strip
"1 ✓ Inspiration · 2 ✓ Direction · 3 Confirm" — Step 3 active. Steps 1 and 2 remain clickable, return there with full state preserved.
Recovery routes
"← Back to adjust" → Step 2 with primary/supporting/prompt/shape preserved. "Use different references" (refusal only) → Step 1, picks preserved but designations cleared. "Try again" → re-fires Generate with the same inputs and returns to Pending. "Save & Confirm" → toast + nav to /design/[id] (or /library per team).
Motion
Pending: 4s nail-fill loop with a 3.6s glimmer sweep over a soft warm gradient. Pending → Success: 250ms cross-fade · result image scales 0.96 → 1.0 in 200ms. Pending → Error: 200ms gentle settle on glyph (NOT shake). prefers-reduced-motion → all loops + transitions become instant; pending falls back to static "Generating…" text + indeterminate bar.
A11y
aria-busy="true" on the result region while pending. Each transition fires an aria-live="polite" announcement ("Generation complete. Result ready." / "Generation paused. Rate limit."). Glyphs are aria-hidden; semantics live in the H2 headline + body. No color-only signaling — every error has glyph + heading + body + ≥1 CTA.

No buttons during pending. Footer reads "No actions while generating." Optional quiet "Cancel" link sits inline above the footer if the team chooses to ship it; flagged for review.
